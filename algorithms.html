Binary Search Algorithm

  • Binary algorithm finds the position of a target value within a sorted array.
  • The binary search algorithm can be classified as a dichotomic divide and conquer search algorithm and executes in logarithmic time.
  • Worst case time complexity O(log n).
  • Best case performance Ω(1).
  • Average case performance Θ(log n).
  • Worst case space complexity O(1).
  • Example

Bubble Sort Algorithm

  • Bubble sort is a simple sorting algorithm that repeatedly steps through the list to be sorted, compares each pair of adjacent items and swaps them if they are in the wrong order.
  • The pass through the list is repeated until no swaps are needed, which indicates that the list is sorted.
  • Worst case time complexity O(n²).
  • Best case performance Ω(n).
  • Average case performance Θ(n²).
  • Example

Bucket Sort Algorithm

  • Bucket sort is a sorting algorithm that works by partitioning an array into a number of buckets.
  • Each bucket is then sorted individually, either using a different sorting algorithm, or by recursively applying the bucket sorting algorithm.
  • Bucket sort can be implemented with comparisons and therefore can also be considered a comparison sort algorithm.
  • Worst case time complexity O(n²).
  • Best case performance Ω(n+k).
  • Average case performance Θ(n+k).
  • Worst case space complexity O(n\cdot k).
  • Example

Insertion Sort Algorithm

  • Insertion sort is a simple sorting algorithm that builds the final sorted array (or list) one item at a time.
  • It is much less efficient on large lists than more advanced algorithms such as quicksort, heapsort, or merge sort.
  • Worst case time complexity O(n²).
  • Best case performance Ω(n).
  • Average case performance Θ(n²).
  • Worst case space complexity O(n).
  • Example

Merge Sort Algorithm

  • Merge sort comparison-based sorting algorithm.
  • Most implementations produce a stable sort, which means that the implementation preserves the input order of equal elements in the sorted output.
  • Mergesort is a divide and conquer algorithm that was invented by John von Neumann in 1945.
  • Worst case time complexity O(nlogn).
  • Best case performance Ω(nlogn).
  • Average case performance Θ(nlogn).
  • Worst case space complexity O(n).
  • Example

Quick Sort Algorithm

  • Quicksort (sometimes called partition-exchange sort) is an efficient sorting algorithm, serving as a systematic method for placing the elements of an array in order.
  • Quicksort can operate in-place on an array, requiring small additional amounts of memory to perform the sorting.
  • Worst case time complexity O(n²).
  • Best case performance Ω(nlogn).
  • Average case performance Θ(nlogn).
  • Worst case space complexity O(n).
  • Example

Radix Sort Algorithm

  • Radix sort is a non-comparative integer sorting algorithm that sorts data with integer keys by grouping keys by the individual digits which share the same significant position and value.
  • A positional notation is required, but because integers can represent strings of characters (e.g., names or dates) and specially formatted floating point numbers, radix sort is not limited to integers.
  • Worst case time complexity O(mn).
  • Example

Selection Sort Algorithm

  • Selection sort is a sorting algorithm, specifically an in-place comparison sort.
  • It has O(n²) time complexity, making it inefficient on large lists, and generally performs worse than the similar insertion sort. Selection sort is noted for its simplicity, and it has performance advantages over more complicated algorithms in certain situations, particularly where auxiliary memory is limited.
  • Worst case time complexity O(n²).
  • Best case performance Ω(n²).
  • Average case performance Θ(n²).
  • Worst case space complexity O(n²).
  • Example
